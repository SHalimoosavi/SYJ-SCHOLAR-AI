## Why I Built This

I've spent enough years writing software to know that most "AI study tools" are solving a problem rich students never had. A subscription here, a cloud API key there, a Wi-Fi requirement baked in by people who never once asked what happens when a student has none of those things.

**SYJ Scholar AI** doesn't make that assumption. Hand it a PDF, and it gives you back summaries, structured notes, flashcards, and full quizzes ‚ÄĒ generated entirely on-device, with an open-source model running inside Termux on an Android phone. No subscription. No cloud round-trip. No data leaving the device, because there's nowhere for it to go.

If you've used "premium" study apps that gatekeep flashcards behind a paywall, you'll understand why this exists.

---

| What students actually face | What Scholar AI does about it |
|---|---|
| A pile of PDFs and not enough hours | Instant chapter-level or full-document summaries |

| No laptop, only a phone | Native Termux/Android-first design, not a port |
| Exam anxiety from not knowing what's coming | AI-generated likely questions and revision checklists |

## Termux Setup ‚ÄĒ Step by Step

```bash
# 1 ‚ÄĒ Install Termux from F-Droid (not the outdated Play Store build)
#     https://f-droid.org/packages/com.termux/

# 2 ‚ÄĒ Update packages
pkg update && pkg upgrade -y

# 3 ‚ÄĒ Install Scholar AI
curl -fsSL https://raw.githubusercontent.com/SHalimoosavi/SYJ-SCHOLAR-AI/main/install.sh | bash

# 4 ‚ÄĒ Install Ollama, the local AI engine
pkg install ollama

# 5 ‚ÄĒ Start Ollama and pull a model sized to your device
ollama serve &
ollama pull gemma:2b      # Recommended ‚ÄĒ fast, ~2GB RAM
# ollama pull phi:mini    # Lighter ‚ÄĒ ~1.3GB RAM
# ollama pull tinyllama   # Minimal ‚ÄĒ ~0.6GB RAM

# 6 ‚ÄĒ Launch
scholar
```

## Pick a Model for Your Hardware

| Model | RAM Needed | Speed | Best For |
|---|---|---|---|
| `gemma:2b` | ~2 GB | ‚ö°‚ö°‚ö° | Recommended default |
| `phi:mini` | ~1.3 GB | ‚ö°‚ö°‚ö°‚ö° | Lightweight devices |
| `tinyllama` | ~0.6 GB | ‚ö°‚ö°‚ö°‚ö°‚ö° | Very low-RAM phones |
| `llama3:8b` | ~5 GB | ‚ö°‚ö° | Higher-quality output |
| `mistral:7b` | ~4.5 GB | ‚ö°‚ö° | Best reasoning |

If you're not sure, start with `gemma:2b`. It's the sane default for a reason.
